The goal of this exercise is to create a class that stores up to N integers and computes:

the shortest span (smallest difference)
the longest span (largest difference)

It also introduces adding a range of iterators to insert many values at once.

---

addNumber()

Adds one integer.

Flow
    addNumber(42)

    Is vector full?
            │
    Yes ─────► throw exception
            │
        No
            │
    push_back(42)

Example
    Span sp(5);

    sp.addNumber(6);
    sp.addNumber(3);
    sp.addNumber(17);

Result
    6 3 17

---

addRange()

Instead of
    sp.addNumber(1);
    sp.addNumber(2);
    sp.addNumber(3);
...

you can do
    std::vector<int> v;

    v.push_back(1);
    v.push_back(2);
    v.push_back(3);

    sp.addRange(v.begin(), v.end());

Flow

    begin ---------- end
    │               │
    ▼               ▼

        1 2 3 4 5

for every iterator
    ↓

    addNumber(*it)

---

longestSpan()

The largest distance between two numbers.

Example
    6 3 17 9 11

Smallest value
    3

Largest value
    17

Longest span
    17 - 3 = 14

Algorithm
    Need at least 2 numbers?

            │
    No ─────► exception
            │
        Yes
            │
    minimum element
    maximum element
            │
    return max - min

Implementation uses
    std::min_element()
    std::max_element()

Complexity
    O(n)

---

shortestSpan()

Find the smallest difference between any two numbers.

Example
    6 3 17 9 11

Sort first
    3 6 9 11 17

Now compare neighbors
    6 - 3  = 3

    9 - 6  = 3

    11 - 9 = 2  ← shortest

    17 - 11 = 6

Result
    2

Why sort?
    Without sorting
        6 3 17 9 11

    You would have to compare
        6 with 3
        6 with 17
        6 with 9
        6 with 11
        ...

        3 with 17
        3 with 9
        ...

    which is
        O(n²)

    -> After sorting
        3 6 9 11 17
    
    The closest numbers must be neighbors.

Only compare
    3-6
    6-9
    9-11
    11-17

Complexity becomes
    Sort      O(n log n)
    Scan      O(n)
    Total     O(n log n)

Why std::vector<int>

1. Dynamic size up to N
    You don't know the numbers beforehand.

    Span sp(10000);

    sp.addNumber(42);
    sp.addNumber(17);
    sp.addNumber(9);

    A vector grows automatically as you call push_back().

2. Fast insertion
    _numbers.push_back(number);

    Adding to the end of a vector is O(1) (amortized), making it ideal for addNumber().

3. Works with STL algorithms

    You need algorithms like:

    std::sort(...)
    std::min_element(...)
    std::max_element(...)

    A vector provides iterators that work perfectly with these algorithms.

4. Supports iterator ranges

    The subject asks you to implement something like:

    sp.addRange(begin, end);

    A vector naturally supports:

    v.begin()
    v.end()

    making range insertion easy.

5. Sorting is efficient

    For shortestSpan() you need to sort.

    std::sort(_numbers.begin(), _numbers.end());

    std::sort requires random-access iterators.

    std::vector provides them.

    std::list does not.

std::runtime_error is a standard exception class in C++.

Think of it as a ready-made object whose job is to report runtime problems.

It lives in:
    #include <stdexcept>

Why do we use it?
    Suppose your Span can only hold 3 numbers.

    Span sp(3);

    sp.addNumber(10);
    sp.addNumber(20);
    sp.addNumber(30);
    sp.addNumber(40);   // Error

Instead of silently failing or crashing, you tell C++:

    throw std::runtime_error("Span is full");

This means:

    "A problem happened while the program was running."

What is runtime_error("Span is full")?

    std::runtime_error("Span is full")

    creates an exception object containing the message:
        Span is full

        Then
            throw std::runtime_error("Span is full");

        throws that object.

    How is it caught?
        try
        {
            sp.addNumber(40);
        }
        catch (const std::exception& e)
        {
            std::cout << e.what() << std::endl;
        }

Here's what happens:
    1. throw creates a runtime_error.
    2. The program leaves addNumber() immediately.
    3. It looks for a matching catch.
    4. catch receives the exception.
    5. e.what() returns the error message.

Output:
    Span is full

Why catch (std::exception&) works
    std::runtime_error inherits from std::exception.

        std::exception
                ▲
                │
        std::runtime_error

    So this catches it:
        catch (const std::exception& e)
        
        because every runtime_error is a std::exception.

---

Other standard exceptions

C++ provides several exception classes:

std::runtime_error
std::logic_error
std::out_of_range
std::invalid_argument
std::overflow_error

For example:
    throw std::out_of_range("Index out of range");


Make own exception:
    class SpanFullException : public std::exception
    {
    public:
        const char* what() const throw()
        {
            return "Span is full";
        }
    };

Then you can write:
    throw SpanFullException();

What is std::runtime_error?

It is simply a class provided by the C++ Standard Library.

class runtime_error : public exception
{
    ...
};

It inherits from std::exception.

std::exception
        │
        ▼
std::runtime_error

Its purpose is to represent an error that happens while the program is running.

For example:
    - file not found
    - divide by zero
    - container is full
    - invalid operation

When you write
    throw std::runtime_error("Span is full");

you are doing two things:
    - Creating a runtime_error object
    - std::runtime_error error("Span is full");
    - Throwing it
    - throw error;
    The shorter version combines them into one line.

Why is it called runtime?
    Because the error is not known when compiling.

Example:
    Span sp(3);

    sp.addNumber(1);
    sp.addNumber(2);
    sp.addNumber(3);
    sp.addNumber(4);

The compiler cannot know you'll add a fourth number.

Only when the program runs does it discover the container is full.

What are min_element() and max_element()?
    They are STL algorithms.

    std::min_element(begin, end);

    returns an iterator pointing to the smallest element.

Example:
    std::vector<int> v;

    v.push_back(6);
    v.push_back(3);
    v.push_back(17);
    v.push_back(9);

Memory:
    6  3  17  9
    ↑
    min_element()

Since it returns an iterator, use * to get the value:
    int minimum = *std::min_element(v.begin(), v.end());

Result:
    minimum = 3

Likewise,
    int maximum = *std::max_element(v.begin(), v.end());

Result:
    maximum = 17
